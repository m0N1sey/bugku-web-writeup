# BugKu Web8 - 头等舱

## 题目信息
- **平台**: [BugKu CTF](https://ctf.bugku.com/)
- **类型**: Web
- **难度**: ⭐⭐
- **金币**: 1
- **分数**: 15

## 题目描述
打开页面后几乎什么都没有，源码也很干净，提示"头等舱"。

## 解题过程

### 1. 打开题目
页面显示空白或简单文字，源码中没有明显线索。

### 2. 抓包分析
"头等舱"暗示要查看 HTTP 头部（Header）信息。

**使用 Burp Suite：**
1. 配置浏览器代理到 Burp Suite（127.0.0.1:8080）
2. 开启 Intercept（拦截）
3. 访问题目页面
4. 在 Proxy → Intercept 中，点击 **Forward** 放行请求
5. 在 Proxy → HTTP History 中找到该请求
6. 点击 **Response** 标签，查看 **Headers**

**或使用浏览器开发者工具：**
1. 按 F12 打开开发者工具
2. 切换到 Network 标签
3. 刷新页面
4. 点击第一个请求
5. 查看 Response Headers

### 3. 发现 Flag
在 HTTP 响应头中找到：flag{f787877a7f0a870b6b44cbefee93678f}

## 涉及知识点
- HTTP 请求头/响应头：Header 中可携带自定义信息
- Burp Suite 抓包：拦截和分析 HTTP 流量
- 浏览器 Network 工具：查看请求和响应详情
