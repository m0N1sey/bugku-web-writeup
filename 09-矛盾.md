# BugKu Web9 - 矛盾

## 题目信息
- **平台**: [BugKu CTF](https://ctf.bugku.com/)
- **类型**: Web
- **难度**: ⭐⭐
- **金币**: 1
- **分数**: 10
- **题目链接**: http://123.206.87.240:8002/get/index1.php

## 题目描述
题目要求传入一个参数 `num`，要求既**不是数字**又**等于 1**，看似矛盾。

## 解题过程

### 1.分析源码逻辑

题目源码如下：

```php
$num=$_GET['num'];
if(!is_numeric($num)) {      // 要求 num 不是数字
    echo $num;
    if($num==1) {            // 但又要求 num 等于 1
        echo 'flag{**********}';
    }
}
```

### 2.利用 PHP 弱类型
**PHP 的 == 是弱比较，会自动转换类型：**
| 表达式                | 结果      | 原因                 |
| :----------------- | :------ | :----------------- |
| `"1a" == 1`        | `true`  | 字符串转数字，取开头数字部分 `1` |
| `"1e0" == 1`       | `true`  | 科学计数法，`1e0` = `1`  |
| `"1.0" == 1`       | `true`  | 浮点数转整数             |
| `is_numeric("1a")` | `false` | 包含字母，不是纯数字         |

### 3.构造 Payload
**在浏览器地址栏输入：**
```
http://123.206.87.240:8002/get/index1.php?num=1a
```
**或：**
```
http://123.206.87.240:8002/get/index1.php?num=1e0.1
```

### 4.获取 Flag
页面返回 flag:flag{5e36e5c90801145b232f4c39e74f1fbf}

## 涉及知识点
- PHP 弱类型比较：== 会自动类型转换，=== 才是严格比较
- is_numeric() 函数：判断是否为数字或数字字符串
- 类型转换规则：字符串转数字时，取开头的数字部分
